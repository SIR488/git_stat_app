**Описание проекта**

•	Название и краткое описание:

\- Система для визуализации активности разработчиков на основе их статистики с GitHub, с возможностью атворизации через GitHub.

\- Когда программисту необходимо продемонстрировать свои навыки, например на собеседованиях, открывать каждый репозиторий и смотреть его статистику – не самый оптимальный вариант. В GitHub отсутствует доступ к сводной статистике по активности разработчика, а также хотелось бы за минимальное количество кликов получать информацию о каждом репозитории, не «путешествуя» по сайту.

•	Стек технологий:

\- Backend: Python 3.13.9, Django 6.0, requests 2.32.5
\- Frontend: JavaScript, Chart.js
\- База данных: SQLite
\- Внешние интеграции: GitHub REST API v3, GitHub OAuth 2.0

•	Установка и запуск:

1\. Конечно необходимо скачать или клонировать репозиторий
для клонирования
```bash
git clone https://github.com/SIR488/git_stat_app.git
```

2\.	Для локального запуска необходимо создать своё OAuth приложение на Github.
Нажимаете на свой профиль -> settings -> в самом низу developer settings -> OAuth Apps -> New OAuth App

Homepage URL
http://localhost:8000

Authorization callback URL
http://localhost:8000/auth/github/callback

остальные данные не несут важности

отсюда нам в будущем понадобится Client ID и Client secrets (сгенерируйте если нет)

3\.	Наконец заходим в ранее установленный проект. Устанавливаем все зависимости из директории /git_stat_app:
```bash
pip install -r requirements.txt
```

4\.	Создаём файл '.env' (тут будут храниться переменные среды) в этой же директории (для навигации: тут у вас есть manage.py)
заполняем его таки образом:
django_secret_key = __тут ключ, который генерируется при создании джанго проекта, его можно сгенерировать__
```terminal
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```
debug=True __Регулирует будут ли высвечивать ошибки на сайте, для локального запуска советую оставить__;
GITHUB_CLIENT_ID = Client ID с гитхаба;
GITHUB_CLIENT_SECRET = Client secrets с гитхаба;
GITHUB_REDIRECT_URI = http://localhost:8000/auth/github/callback __куда гитхаб будет нам пересылать__;

5\. Запускаем миграции (должен создасться файл db.sqlite3 в той же директории)
```terminal
python manage.py migrate
```

6\. Заходим на один уровень глубже в проект, в папку git_stat_app и выполняем команду в терминале (прошлую команду можно запустить и здесь)

```terminal
python manage.py runserver
```

7\. Всё готово
Можете переходить в браузере по ссылке http://127.0.0.1:8000/
